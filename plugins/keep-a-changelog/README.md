# Keep a Changelog

Keep a Changelog 1.1.0 format guide for creating, updating, and maintaining CHANGELOG.md files.

## Overview

This plugin provides guidance on the Keep a Changelog 1.1.0 specification, helping you maintain a
curated, chronologically ordered list of notable changes for each version of a project. It covers
all six standard section types (Added, Changed, Deprecated, Removed, Fixed, Security), version
comparison links, yanked releases, release cutting, and the relationship with Semantic Versioning.

## Skills

### Auto-Loaded

**Keep a Changelog**

Activates when you create, edit, or review CHANGELOG.md files, cut releases, manage `[Unreleased]`
sections, or format changelog entries. Provides guidance on changelog structure, entry formatting,
version headers, and comparison link generation.
