# Domain-Driven Design

Strategic and tactical DDD patterns for modeling complex business domains.

## Overview

This plugin provides comprehensive guidance for Domain-Driven Design, covering both
strategic patterns (bounded contexts, subdomains, context mapping) and tactical patterns
(aggregates, entities, value objects, domain events). It supports multiple implementation
styles: traditional OOP, functional Decider pattern, event sourcing, CQRS, and
actor-based systems.

## Skills

### Auto-Loaded

**domain-driven-design**

Activates when you work on domain modeling, aggregates, event sourcing, CQRS, bounded
contexts, or complex business logic. Provides pattern guidance, code examples, and
architectural decision support for .NET/C# and Python implementations.
